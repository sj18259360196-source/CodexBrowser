# Package version 0.1.1

## Objective

Package the latest Codex Browser source, including browser-native skill learning and management, as a new Windows installer with an updated application version.

## Decisions

- Use the next patch version, `0.1.1`, because the previous package was `0.1.0` and this build adds backward-compatible functionality.
- Keep the existing x64 assisted NSIS installer configuration, bilingual installer resources, application icon, tray icon, and branded installer artwork.
- Run runtime smoke checks with isolated named pipes and user-data directories so verification does not touch the normal browser profile.
- Preserve the user's concurrent public README, license, security, branch, and release-preparation changes without rewriting or reverting them.

## Changes

- Updated `package.json` and `package-lock.json` from version `0.1.0` to `0.1.1`.
- Rebuilt the renderer, Electron desktop, browser-skill service, and MCP server.
- Generated `release/Codex-Browser-Setup-0.1.1-x64.exe`, its block map, update metadata, and the unpacked Windows application.
- Rebuilt after the concurrent public-release metadata update so the final package includes the current author, MIT license, repository, homepage, and issue-tracker metadata.
- Added this session record and linked it from `WORKLOG.md`.
- No personal plugin source, installed plugin record, or plugin cache files were changed during this packaging session.

## Verification

- `npm run typecheck` passed.
- `npm run build` passed for the renderer, Electron desktop, browser-skill service, and MCP server.
- `npm run smoke:skills` passed service learning, persistence, matching, statistics, and import rejection checks.
- `npm run smoke:skills-runtime` passed against an isolated profile and verified semantic execution, draft learning, and removal of raw filled values.
- `npm run smoke:mcp` passed against an isolated profile and reported 38 tools with protocol `1.3.0`.
- `npm run smoke:actions`, `npm run smoke:runtime`, `npm run smoke:credentials`, and `npm run smoke:advanced` passed.
- Playwright verified the open skill drawer at 1440 x 900 and the learning view at 720 x 900 with zero console errors. Screenshots are `output/playwright/package-0.1.1-skills-desktop.png` and `output/playwright/package-0.1.1-skills-narrow.png`.
- The installer metadata reports product version `0.1.1`, file version `0.1.1`, and the current package author as its company name.
- The unpacked application reports product version `0.1.1.0`, contains the packaged branding resources, and successfully started with an isolated profile.
- The packaged runtime returned protocol `1.3.0`, 23 capabilities, and the `browser-skill-learning` capability through its named pipe.
- `release/latest.yml` points to `Codex-Browser-Setup-0.1.1-x64.exe` with matching size and checksum metadata.

## Known issues

- The installer and application are not digitally signed, so Windows SmartScreen may warn users until a code-signing certificate is configured.
- The generated `release/` directory is intentionally ignored by Git and remains a local build artifact until a release upload is requested.
- The repository working tree contains substantial uncommitted changes from the feature, branding, documentation, public-release, and packaging sessions.

## Next steps

- Install the `0.1.1` package on a clean Windows user profile for a manual first-run and uninstall check.
- Configure Authenticode signing before wider distribution.
- Commit the intended source and work-log changes, then publish the installer as a GitHub Release when requested.
