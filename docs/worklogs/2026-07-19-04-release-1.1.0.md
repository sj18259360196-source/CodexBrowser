# Release 1.1.0 Packaging And Publication

## Objective

Package the completed ordinary Edge extension relay and challenge/PDF compatibility fixes as Codex Browser 1.1.0, verify the distributable, and publish the source and Windows artifacts to GitHub.

## Decisions

- Use minor version `1.1.0` because the release adds a new opt-in browser runtime while preserving `external-edge` as the default.
- Keep generated ZIP and checksum files outside Git history and upload them as GitHub Release assets.
- Publish source from an `agent/edge-extension-relay-1.1.0` branch through a draft pull request rather than committing directly to `main`.
- Create a versioned GitHub Release only after the source branch, checks, portable-package smoke test, and checksum have succeeded.

## Changes

- Updated application and lockfile versions to 1.1.0.
- Added 1.1.0 release notes and updated README release metadata.
- Made the packaging script include the release-notes file matching the package version.

## Verification

- `npm run typecheck`: passed.
- `npm run build`: passed as part of packaging.
- `npm run test:runtime-selection`: passed, 2/2 tests.
- `npm run test:edge-relay`: passed, 1/1 test.
- `npm run test:challenge`: passed, 19/19 tests.
- `npm run package:win`: passed and generated `CodexBrowser-1.1.0-win-x64.zip` plus its SHA-256 file.
- `npm run smoke:release-package`: passed from the extracted archive on Edge 150.0.4078.65; the packaged broker and MCP were present, the isolated Profile reached ready, and shutdown released runtime resources.
- Final archive size: 180,710,551 bytes. SHA-256: `58fbb665c9643e435cd9b5de3090a75ee00b432f77674d50da602741da09bcba`.
- Source commit `c5f5373` was pushed to `agent/edge-extension-relay-1.1.0`.
- Draft PR #2 was opened against `main`: `https://github.com/sj18259360196-source/CodexBrowser/pull/2`.
- Draft GitHub Release `v1.1.0` was created from the release branch. GitHub reports both the 180,710,551-byte ZIP and checksum asset in `uploaded` state, and independently reports the ZIP digest matching the local SHA-256.

## Known Issues

- The portable archive is not code-signed and remains a ZIP rather than a Windows installer.
- Live extension loading, debugger approval, and site verification remain user-operated.
